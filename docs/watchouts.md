# Watchouts — known pitfalls Claude must actively check for

Purpose: every trap identified during project design, in one place. When
working on ANY task in this repo, scan the relevant section first. If you
observe one of these conditions, STOP and flag it to the user before
proceeding — do not silently work around it.

## Experiment validity (can silently corrupt the science)

- **Silent context truncation (the #1 artifact).** If a prompt exceeds
  `num_ctx`, Ollama truncates from the FRONT — which is where the constraint
  lives — fabricating a fake cliff. Check: `prompt_tokens` must grow
  linearly with `padding_level` in every run. If it plateaus, the run is
  invalid. Raise `num_ctx` and rerun; never analyze a plateaued run.
- **T50, not "first zero".** The cliff is the logistic-fit 50% crossing,
  never the first padding level with zero successes. Single-run zeros are
  sampling noise near the transition.
- **One trial per cell is noise.** Any cell with n < 5 trials is
  underpowered — flag it, never report it as a rate without the n.
- **Padding is a variable, not filler.** Never swap padding text casually;
  lorem vs prose vs distractor are separate experimental arms and must
  never be pooled in analysis. Same for constraint position (before/after).
- **Tokenizers differ across models.** Always group/plot by actual
  `prompt_tokens` (from `prompt_eval_count`), never by `padding_level` or
  word-count estimates, when comparing models.
- **Right-censor, never extrapolate.** If success never drops below 50% in
  the tested range, report "T50 > max tokens tested". Extrapolated T50s
  are fabrications.
- **Non-determinism invalidates runs.** Every call: temperature 0.0 +
  explicit seed. If any result can't be reproduced, check the CPU
  determinism audit before trusting it — threading on CPU can break
  bit-exact reproduction even at temp 0.
- **Query/constraint interaction.** For `negative_the`, a query whose
  natural answer forces the forbidden word measures the query, not the
  constraint. Check query wording whenever adding queries or constraints.
- **JSON validator leniency is a documented choice.** Markdown fences are
  tolerated so we measure JSON-ness, not fence discipline. If tightened,
  it's a new experiment version — old results not comparable.
- **Refusal-boilerplate collision (observed 2026-07-08, `negative_the` +
  `prose`, levels 96/160, both `llama3.2:3b` and `llama3.2:1b`).** At high
  padding levels the model can mis-frame the task as reading-comprehension
  ("this question is not related to the provided text") and refuse to
  answer, instead of forgetting the constraint while answering normally.
  For `negative_the` specifically, the refusal boilerplate itself often
  contains "the" ("not related to **the** provided text"), which is what
  trips the validator — not word-choice decay in a genuine answer. When
  the refusal phrasing happens to avoid "the", it passes despite still
  being a refusal. Treat "constraint violated" and "task misframed" as
  two different underlying phenomena when writing up results from this
  cell: check `output_snippet` for refusal language, don't assume every
  `success=0` row is the model losing track of the rule mid-answer.

## Reproducibility & data integrity

- **`data/raw/` is immutable.** Never edit, overwrite, delete, or "clean
  in place". Derived data -> `data/processed/` via a committed script.
- **Validator/instruction changes = version bump.** Any change to a
  validator, an instruction string, the seed formula
  (`random.Random(seed * 7919 + level)`), or the CSV column order breaks
  comparability. Bump `EXPERIMENT_VERSION`, log in docs/changelog.md.
  CSV schema is append-only.
- **Crash-safety.** Runner must flush after every row and record failures
  as `success = -1` (excluded from fits, exclusion count reported).
  Never let an exception kill a multi-hour run.

## Hardware & runtime (32GB RAM, CPU-only)

- **Never kill or relaunch a running experiment without asking.** Runs
  take hours; a restart discards wall-clock, not just data.
- **Never run two models in parallel.** Ollama will swap; timing data
  becomes garbage and the box may OOM.
- **fp16 12B is borderline (~24GB).** Run it alone with nothing else open;
  prefer q8_0 if the machine has other load. Warn before suggesting fp16.
- **Budget the grid before launching.** models x constraints x conditions
  x levels x trials — estimate call count and hours (12B on CPU: minutes
  per long-context call, 600s timeouts). Suggest nohup/tmux for anything
  over ~30 min.
- **Verify model tags exist** (`ollama list`) before writing commands —
  never assume a tag or quant variant is pulled.

## Analysis & statistics

- Bootstrap CIs resample at the trial level, >= 1000 resamples, seed=42.
- Exclude error rows (`success == -1`) and report how many were excluded.
- Figures are script-generated only; if a figure can't be regenerated by
  one command, it doesn't go in the paper.
- Calibration study: verbalized confidence comes from the json_schema
  constraint output — validate the field parses as a number in [0,1]
  before computing ECE; malformed confidences are exclusions, not zeros.

## Paper & claims

- No number in the draft without a matching value in `results/fits/` or a
  figure. Placeholders: `[T50: TBD]`, `[CITATION NEEDED: topic]`.
- Scope claims to tested models/sizes/constraints; no "all LLMs".
- Report null results (constraints that never decayed) prominently.
- Injection study (Study 3) is measurement/defensive framing ONLY:
  characterize susceptibility of local models to instruction-shaped text
  in context. Do not generate, optimize, or iterate on attack payloads
  beyond the four fixed distractor levels defined in the agenda.

## Claude Code workflow

- Read the nested CLAUDE.md of the directory you're editing; run
  `pytest experiments/tests/ -x -q` after touching validators or padding.
- When two statistical/design approaches are defensible, present both and
  let the user choose — methodology decisions affect the paper's claims.
- Minimal diffs; no refactors of untouched code; one logical change per
  commit; never commit `.venv/` or raw CSVs > 50MB.
